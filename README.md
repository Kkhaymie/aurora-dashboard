## Overview

An interactive, AI-powered executive dashboard built for Aurora Retail & Digital Services. It transforms 4 years of raw transaction data (2021–2024) into real-time business intelligence — featuring dynamic KPI cards, interactive Plotly charts, and an AI-generated executive briefing powered by the Mistral API.

Designed for non-technical executives who need clear, actionable insights without navigating raw data.

## Live KPIs

| Metric | Value |
|---|---|
| Total Revenue | $6.50M |
| Total Profit | $1.95M |
| Profit Margin | 30.0% |
| Budget Variance | +0.7% |
| Churn Rate | 26.2% |

> **Note on deduplication:** The Aurora dataset contains 20,000 rows — each transaction is repeated 4× (one per department: Sales, Marketing, Operations, IT). All revenue and profit metrics use `drop_duplicates(subset='transaction_id')` to work on 5,000 unique transactions. Budget metrics intentionally use the full 20,000 rows since each row represents one department's budget entry.

## Features

### 1. Dynamic KPI Cards
Five cards with colour-coded accent bars (blue → revenue, green → profit, purple → margin, amber → budget, red → churn). Values update instantly when sidebar filters are applied.

### 2. Interactive Charts
- **Revenue by Region** — horizontal bar chart (North, South, East, West)
- **Revenue by Channel** — horizontal bar chart (Online, In-Store, B2B)
- **Profit by Customer Segment** — bar chart (Consumer, SME, Enterprise)
- **Churn vs Retained** — donut chart with centre annotation showing live churn %
- **Monthly Revenue Trend** — area line chart spanning Jan 2021 – Dec 2024

### 3. AI Executive Briefing
Click **⚡ Generate AI Executive Summary** to send live KPI data to the Mistral API. Returns a 3-paragraph executive narrative covering:
- Overall performance assessment
- Key risks and concerns
- Recommended actions

All in plain, jargon-free language suitable for a non-technical audience.

### 4. Sidebar Filters
Filter by **Region**, **Channel**, **Customer Segment**, and **Year**. All KPI cards and charts update dynamically — no page reload required.

## Local Setup

```bash

cd aurora-dashboard

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set your Mistral API key
```

## Deployment (Streamlit Cloud)

1. Push all files to a public GitHub repository
2. Go to [share.streamlit.io](https://share.streamlit.io) → **New app**
3. Select repo → set main file to `dashboard.py`
4. Go to **Advanced settings → Secrets** and add:
   ```
   MISTRAL_API_KEY = "your_mistral_key_here"
   ```
5. Click **Deploy**

## Assumptions & Design Decisions

- **Deduplication:** Revenue/profit metrics deduplicate on `transaction_id` since each transaction appears 4× in the raw data. Budget metrics use the full dataset intentionally.
- **Date parsing:** `transaction_date` and `signup_date` are stored as proper date strings in the CSV — parsed directly with `pd.to_datetime()` without Excel serial number conversion.
- **Churn definition:** A customer is counted as churned if `churn_flag == 'Yes'`. The `churn_date` column is intentionally null for non-churned customers — this is expected behaviour, not a data quality issue.
- **Budget variance:** Calculated as `(actual_revenue - budgeted_revenue) / budgeted_revenue × 100` across all 4 departments.
